## рҹӨқ Design Principles

1. **DRY (Don't Repeat Yourself)**
   - Semua reusable logic di ModuleScript

2. **Single Responsibility**
   - Setiap script punya 1 tanggung jawab

3. **Clear Naming**
   - Grid convention (B1C1, B1C2)
   - Descriptive function names

4. **Server Authority**
   - Server handles critical logic (teleport)
   - Prevent client-side cheating

5. **Scalability**
   - Mudah add bamboo baru (tinggal add ke folder)
   - Mudah add more teleporters
